music-website项目配置

开源地址：Yin-Hongwei/music-website: 🎧 Vue + SpringBoot + MyBatis 音乐网站

初始配置：

JDK： jdk-8u141

mysql：mysql-5.7.21-1-macos10.13-x86_64（或者更高版本）

redis：5.0.8 或 docker启动redis | 想飞跃的鱼 (nanshaws.github.io)

node：14.17.3 【不要超过大版本16】

IDE：IntelliJ IDEA 2018、VSCode

minio: 下载本地最新 或者 docker完美启动minio | 想飞跃的鱼 (nanshaws.github.io)

克隆源码问题：

git clone https://github.com/Yin-Hongwei/music-website.git

Failed to connect to github.com port 443 after 21054 ms: Could not connect to server

一般出现这种问题都是开过VPN导致的本机系统端口号和git的端口号不一致导致的

Git默认使用SSH协议，其默认端口是22

HTTPS协议默认使用端口443

解决方法：

  1. 查看本机系统端口号

设置->网络和Internet->代理

  1. 设置git端口号和上面的端口号保持一致
  1. 查看git设置

git config –global -l

user.name=Inf
user.email=2035236465@qq.com
http.proxy=127.0.0.1:7890
https.proxy=127.0.0.1:7890

  1. 如果没有挂着vpn，但是还是遇到了以上报错

还是一样的原因，两边的端口不一致导致的，那么直接去掉git代理即可：

git config –global –unset http.proxy

git config –global –unset https.proxy

minio安装配置：

  1. 安装与配置Minio dl.min.io/server/minio/release/windows-amd64/ cmd打开mini.exe所在文件夹
  2. 设置管理员账号和密码

set minio_root_user=minioadmin

set minio_root_password=minioadmin

  1. 执行以下指令启动minio

minio.exe server D:\MinIo\minio –console-address “:9001” –address “:9000”

这条命令用于启动MinIO服务器，指定了数据存储的路径为P:\MinIo\minio，并设置了MinIO服务的API端口为9000，控制台端口为9001

  1. 然后就是访问 localhost:9001 创建bucket
  2. 单服务配置文件添加minio配置

minio:
endpoint: http://xxx.xxx.xxx.xxx:9000
accessKey: minioadmin
secretKey: minioadmin
bucketName: minioadmin

  1. 修改单服务配置文件 文件大小限制（默认1MB）

spring:
servlet:
multipart:
max-file-size: 5MB
